Chrome Extension Setup Steps

Step 1: Load Extension in Chrome

  1. Open Chrome and go to chrome://extensions/
  2. Enable "Developer mode" (top right toggle)
  3. Click "Load unpacked"
  4. Select the dist folder from your extension project
  5. Copy the Extension ID that appears under your extension
Extension ID Format: Something like abcdefghijklmnopqrstuvwxyzabcdef

Step 2: Create OAuth Client in Google Cloud Console

  1. Go to Google Cloud Console
  2. Select your project: videoprompter-d6a18
  3. Navigate to APIs & Services > Credentials
  4. Click + CREATE CREDENTIALS > OAuth client ID
  5. Select Chrome Extension as application type
  6. Enter your Extension ID from Step 1
  7. Click Create
  8. Copy the generated Client ID
Client ID Format: 123456789-abcdefghijk.apps.googleusercontent.com

Step 3: Update manifest.json

Replace the client_id in dist/manifest.json:

"oauth2": {
  "client_id": "YOUR_ACTUAL_CLIENT_ID_HERE.apps.googleusercontent.com",
  "scopes": [
    "https://www.googleapis.com/auth/userinfo.email",
    "https://www.googleapis.com/auth/userinfo.profile"
  ]
}

Step 4: Test Sign In

  1. Go back to chrome://extensions/
  2. Click the refresh icon on your extension
  3. Click the extension icon in the toolbar
  4. Click the "ðŸ”‘ Sign In" button
  5. Check the browser console (F12) for any error messages
Console Check: Look for messages like "Sign in button clicked" and any error messages

Common Issues & Solutions

  • "Chrome Identity API not available" - Extension permissions issue
  • "Invalid client" - Wrong Extension ID or Client ID
  • Nothing happens - Check console for JavaScript errors
  • "Unauthorized" - Need to configure OAuth consent screen
